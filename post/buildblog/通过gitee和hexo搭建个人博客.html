通过gitee和hexo搭建个人博客

发表于 2021-11-29 分类于 个人博客 阅读次数： 本文字数： 2.1k 阅读时长 ≈ 2 分钟

首先准备软件：

  • git (提供命令git) git官网
  • notepad++(方便编辑)notepad++官网
  • nodejs(hexo依赖)nodejs官网
  • 7z(压缩包)7z官网

还有码云主页

安装

nodejs、git、notepad++和7z的安装过程省略。主要要说的是hexo的安装和配置。
再以上几个安装好后，注意测试是否可以再命令行可以使用：git,npm
这里虽然使用的是Windows值作但是全部都是使用命令完成，与Linux的命令相同，所以可以Linux也是可以完成的。
Win+R-> ‘cmd’ 打开命令行
首先需要配置npm的国内源

npm config set registry https://registry.npm.taobao.org		#配置国内的镜像源
npm info hexo	# 测试查看hexo的安装信息，是否是taobao.org的源

然后再是安装hexo：

npm install -g hexo 		# 通过npm安装hexo
# -g 指定全局安装，可以使用hexo命令

在需要保存自己博客’/Myblog/‘的路径下执行下面操作

hexo init test		# 初始化创建，会创建test文件夹/Myblog/test/
cd test			# 进入Myblog目录
npm install			# 进一步安装hexo所需文件

新建完成后，指定文件夹目录下有：

  • node_modules: 依赖包
  • public: 存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • _config.yml: 博客的配置文件

这样hexo就安装完成了，接下来就可以启动hexo了

hexo clean			# 清除所有记录
hexo g		# 生成静态网页
hexo s	# 启动服务

然后使用浏览器访问http://localhost:4000/

创建Gitee的仓库

gitee主页
再gitee创建自己的账户，然后再创建一个自己仓库

在创建仓库完成后进入到仓库

然后在Myblog下打开git bash进行clone

git clone https://gitee.com/NeuronBear/RongkangXiongBlog.git

然后将/Myblog/test刚刚的文件除了.github文件以外都复制到/Myblog/你的克隆下来的仓库文件下，之后你可以将test文件夹整个删除（后面没啥用了）

然后cd进入/Myblog/test目录下

你可以重新启动服务

hexo s

然后打开浏览器http://localhost:4000/查看生成的页面

现在将更新的文件上传到gitee

之后的/blog指的是你的clone下来的文件夹的名称

复制URL，到hexo的网站配置文件/blog/_config.yml的最后

……
deploy:
  - type: git    # type为git
    repo: git@gitee.com:NeuronBear/RongkangXiongBlog.git  # 仓库的URL
    branch: master   #构建网页的分支
……

这里在gitee上再创建一个除了master外的分支hexo用来放源码

这里先安装一个hexo的插件

npm install hexo-deployer-git --save	# 安装git插件

然后上传文件到仓库

git commit -m "test push"
git remote add origin_gitee https://gitee.com/NeuronBear/RongkangXiongBlog.git #链接到gitee,默认是origin,但为了之后不冲突要和github区分
git push -u origin master

然后就可以看到push上去了

git config --global user.email *********@qq.com	# 设置gitee邮箱（gitee的注册邮箱）
git config --global user.name '****'			# 设置用户名（git的注册昵称）
hexo deploy	# 上传到gitee
# 在上传时，需要再次输入gitee的注册邮箱作为username，账户密码作为password

上传完成之后，仓库就会多出以下文件

主题配置

如果不喜欢这个主题可以换一个
比如next主题，next官网 next的github网页，去github下载压缩包然后放到test下的themes目录下。

然后配置_config.yml

然后配置_config.yml

……
theme: next		# 这里需要注意:后面要有一个空格，名称要和theme下的主题目录名称相同。
……

hexo的其他配置参考hexo官方文档
next的配置参考next官方文档

然后重启hexo服务

hexo clean			# 清除所有记录
hexo generate		# 生成静态网页
hexo server -p 80	# 启动服务

然后重新访问网页，就会发生变化
